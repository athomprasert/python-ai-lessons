Python Essentials for API Work

The six patterns you'll use in almost every Python script you write.

Why this lesson

Before we call APIs or build AI agents, we need a firm grip on six Python patterns that appear everywhere in that work. This lesson is practical — by the end you'll have a real script that prints structured data, not "Hello, World!".

Connection to your mission
Every AI agent you'll ever build calls APIs and processes JSON. This lesson builds exactly those muscles.

1 — Variables & types

Python is dynamically typed — you don't declare types, Python figures them out.

name    = "Attapol"       # str
count   = 42              # int
price   = 9.99            # float
active  = True            # bool
nothing = None            # null / no value

print(type(name))         # <class 'str'>
Tip
None in Python is like null in JSON or null in Java. You'll see it constantly when a field is missing in an API response.

Quick check — Variables

What is the type of x = "42"?

  • int (integer)
  • str (string)
  • float
  • bool

2 — Dictionaries

A dictionary is a key-value store — exactly like a JSON object. It's the most important data structure for API work.

# Create a dict (looks just like JSON)
user = {
    "name":  "Attapol",
    "role":  "Architect",
    "active": True
}

# Read a value
print(user["name"])       # Attapol

# Safe read — returns None if key missing
print(user.get("email"))  # None

# Add or update a key
user["email"] = "a@example.com"

# Loop over keys and values
for key, value in user.items():
    print(key, "→", value)
Tip
Always use .get() instead of [] when reading from an API response — API fields can be missing, and [] will crash while .get() returns None.

Quick check — Dictionaries

What does user.get("phone") return if "phone" is not a key in user?

  • An empty string ""
  • A KeyError exception
  • None
  • False

3 — Lists & loops

A list is an ordered collection. API responses often return a list of objects (e.g. a list of assets in Anypoint Exchange).

apis = ["orders-api", "products-api", "customers-api"]

# Loop over every item
for api in apis:
    print("Found:", api)

# Filter: only APIs whose name contains "orders"
order_apis = [a for a in apis if "orders" in a]
print(order_apis)   # ['orders-api']

# Count
print(len(apis))      # 3

The [a for a in apis if ...] pattern is called a list comprehension — a compact way to filter or transform a list. You'll use this a lot.

4 — Functions

A function groups code you want to reuse. In agent work, you'll write many small functions — one per "action" the agent can take.

# Define a function
def greet(name, role="User"):
    return f"Hello, {name}. You are a {role}."

print(greet("Attapol", "Architect"))
# Hello, Attapol. You are an Architect.

print(greet("Bot"))
# Hello, Bot. You are a User.
Tip — f-strings
f"Hello, {name}" is an f-string (formatted string). Put f before the quote and use {variable} inside. Way cleaner than string concatenation.

5 — JSON: reading & writing

JSON is the language of APIs. Python's built-in json module converts between JSON strings and Python dicts/lists.

import json

# JSON string → Python dict
raw = '{"name": "Products API", "version": "2.0"}'
data = json.loads(raw)
print(data["name"])     # Products API

# Python dict → JSON string (pretty-printed)
payload = {"status": "ok", "count": 3}
print(json.dumps(payload, indent=2))
Remember
json.loads() = "load string" → Python object
json.dumps() = "dump to string" → JSON string
The s stands for string.

6 — Importing libraries

Python ships with a standard library (like json, os). Third-party libraries are installed with pip. Almost all AI work uses third-party libraries.

# Standard library — no install needed
import json
import os

# Import just one thing from a module
from datetime import datetime
print(datetime.now())

# Third-party — install first: pip install requests
import requests      # for HTTP calls (Lesson 2)

Put it together — your first real script

This script simulates what happens when you get a JSON response from an API and want to process it. It uses all six patterns above.

# api_demo.py  ← save this file and run it with: python api_demo.py
import json

# Simulated API response (a JSON string)
response_text = '''
[
  {"name": "orders-api",    "version": "3.1", "status": "active"},
  {"name": "products-api",  "version": "2.0", "status": "active"},
  {"name": "payments-api",  "version": "1.0", "status": "deprecated"}
]
'''

def get_active_apis(json_text):
    all_apis = json.loads(json_text)
    return [api for api in all_apis if api.get("status") == "active"]

def print_summary(apis):
    print(f"Active APIs: {len(apis)}")
    for api in apis:
        print(f"  • {api['name']}  (v{api['version']})")

active = get_active_apis(response_text)
print_summary(active)

Expected output:

Active APIs: 2
  • orders-api  (v3.1)
  • products-api  (v2.0)

Final check — pick the right tool

Q1. An API response includes a field "owner" that may or may not be present. What's the safest way to read it from a dict d?

  • d["owner"]
  • d.get("owner")
  • d.owner
  • d.find("owner")

Q2. You receive a JSON string from an API. Which call converts it to a Python dict?

  • json.loads(text)
  • json.dumps(text)
  • json.parse(text)
  • json.decode(text)

Q3. You have a list items and want a new list containing only items where item["active"] == True. Which is the most Pythonic approach?

  • Write a for loop, check each item, append to a new list manually
  • Use items.filter(lambda x: x["active"])
  • [item for item in items if item["active"]]
  • items.select("active")
