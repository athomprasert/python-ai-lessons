Calling HTTP APIs with Python

Use the requests library to call a real API — GET data, read JSON, handle errors.

Lesson 1 — Essentials ✓ Lesson 2 — HTTP APIs Lesson 3 — Calling Claude Lesson 4 — Building an agent

Why this lesson

Every AI agent talks to the outside world via HTTP — whether that's an Anypoint API, a database, or Claude itself. This lesson gives you one skill: make a real HTTP request from Python and do something with the response.

Connection to your mission
A RAG system pulls documents from a source (often an API). An agent calls tools via HTTP. This pattern — request → parse JSON → act on data — is the skeleton of everything you'll build.

1 — What happens in an HTTP call

When your script calls an API, four things happen in sequence:

  1. Request — your code sends a verb + URL + optional headers/body
  2. Server processes — the API does its work
  3. Response — server sends back a status code + body (usually JSON)
  4. You parse it — convert the JSON body into a Python dict
If you know Java
This is the same as HttpClient or RestTemplate in Java — send a request, get a response, parse the body. Python's requests library just makes it much shorter.

HTTP verbs you'll use most

VerbUse forHas body?
GETRead data — list APIs, fetch a documentNo
POSTCreate something, or send data to processYes (JSON)
PUTReplace a resourceYes (JSON)
DELETERemove a resourceUsually no

Status codes to know

CodeMeaning
200OK — request succeeded, data in body
201Created — POST succeeded, resource created
400Bad request — your payload/params are wrong
401Unauthorized — missing or invalid auth token
404Not found — wrong URL or resource doesn't exist

3 — Anatomy of a GET request

Here's the full pattern — read it once, then we'll break it down:

import requests

response = requests.get("https://api.example.com/products")

# 1. Check the status code
print(response.status_code)    # 200

# 2. Parse the JSON body into a Python dict/list
data = response.json()

# 3. Work with the data
print(data)
response = requests.get("https://api.example.com/products")
↑ the full response object ↑ verb ↑ URL

response.status_code → 200
↑ HTTP status number

response.json() → Python dict or list
↑ parses the body — same as json.loads(response.text)

4 — Adding query parameters

Query parameters are the ?key=value pairs in a URL. Pass them as a dict — requests handles the URL encoding for you.

params = {
    "status": "active",
    "limit":  10
}

response = requests.get(
    "https://api.example.com/products",
    params=params
)

# requests builds: https://api.example.com/products?status=active&limit=10
print(response.url)  # confirms what was sent

5 — Headers & authentication

Most real APIs require a token in the request header. Pass headers as a dict:

headers = {
    "Authorization": "Bearer YOUR_TOKEN_HERE",
    "Content-Type":  "application/json"
}

response = requests.get(
    "https://anypoint.mulesoft.com/exchange/api/v2/assets",
    headers=headers
)
Never hardcode tokens
Real tokens belong in environment variables, not in your source code. We'll cover os.environ and .env files in the next lesson when we call Claude for the first time.

6 — Checking for errors

requests does not raise an exception on 4xx or 5xx responses — it just returns the response with a bad status code. You have to check.

response = requests.get("https://api.example.com/products")

# Option A: check manually
if response.status_code == 200:
    data = response.json()
else:
    print(f"Error: {response.status_code}", response.text)

# Option B: raise_for_status() throws on 4xx/5xx
response.raise_for_status()   # throws HTTPError if not 2xx
data = response.json()
Tip
Use raise_for_status() as your default. It turns a silent failure (bad status code, no exception) into a loud one (exception you can catch). Loud failures are easier to debug.

Put it together — calling a real public API

We'll use JSONPlaceholder — a free, no-auth test API that returns realistic-looking JSON. No account needed.

# call_api.py
import requests
import json

def get_posts(user_id):
    response = requests.get(
        "https://jsonplaceholder.typicode.com/posts",
        params={"userId": user_id}
    )
    response.raise_for_status()
    return response.json()     # returns a list of dicts

def summarise(posts):
    print(f"Found {len(posts)} posts")
    for post in posts:
        print(f"  [{post['id']}] {post['title']}")

posts = get_posts(user_id=1)
summarise(posts)

Expected output (first few lines):

Found 10 posts
  [1] sunt aut facere repellat provident occaecati
  [2] qui est esse
  ...

  3. Modify: Change user_id=1 to user_id=2 — do you get different posts?

Quick check

Q1. requests.get(url) raises an exception if the server returns 404.

  • True
  • False — you must check status_code or call raise_for_status()

Q2. You want to pass ?limit=5&page=2 in a GET request. What's the right approach?

  • Manually append "?limit=5&page=2" to the URL string
  • Add them to the headers dict
  • Pass params={"limit": 5, "page": 2} to requests.get()
  • Use a POST request instead

Q3. An API returns 401. What does that most likely mean?

  • The server is down
  • Your auth token is missing or invalid
  • The URL is wrong
  • Your JSON payload has a syntax error

Bonus — making a POST request

POST sends data to the server. You pass the body as a dict via json= (requests sets the Content-Type header automatically).

payload = {
    "title":  "New API Design",
    "body":   "RAG patterns for MuleSoft",
    "userId": 1
}

response = requests.post(
    "https://jsonplaceholder.typicode.com/posts",
    json=payload             # dict → JSON body + correct Content-Type header
)

response.raise_for_status()
print(response.status_code)  # 201 Created
print(response.json())       # server echoes back the created resource with an id
