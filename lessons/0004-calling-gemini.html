Your First Call to Gemini

Send a prompt to Google Gemini and print the response — the foundation of every agent and RAG system you'll build.

Lesson 1 ✓ Lesson 2 ✓ Lesson 3 ✓ Lesson 4 — Calling Gemini Lesson 5 — System prompts & multi-turn

Why this lesson

Up to now you've called normal HTTP APIs. Calling an LLM is similar — but most teams use the official SDK instead of raw HTTP because it handles auth, retries, and response parsing for you.

Connection to your mission
A RAG system has three parts: retrieve documents → build a prompt → call the LLM. This lesson teaches the third part — the simplest, most repeatable building block.

1 — Get a free API key

Google's free Gemini API tier is generous and doesn't require billing setup. Source: official quickstart.

1 Open aistudio.google.com and sign in with a Google account.
2 Click "Get API key" in the left sidebar.
3 Click "Create API key". Copy the value — it starts with AIza...
4 Treat it like a password. We'll put it in an environment variable, never in source code.

2 — Store the key safely

The Gemini SDK reads the key from an environment variable named GEMINI_API_KEY. Set it once for your shell session:

# In your terminal, in this folder:
export GEMINI_API_KEY="AIza...your-key-here"
Important
export only lasts for the current terminal window. To make it permanent, add the line to ~/.zshrc. Never commit the key to git — if you ever paste it into a file, also add that file to .gitignore.
Better: a .env file (we'll use this in Lesson 5)
Real projects keep keys in a .env file (git-ignored) and load them with the python-dotenv library. For today, export is enough.

3 — Set up a virtual environment, then install the SDK

The current official package is google-genai. (Don't confuse it with google-generativeai — the older legacy package.) Source: SDK README.

Why we don't just run pip install globally
On macOS with Homebrew Python, running pip3 install google-genai will fail with error: externally-managed-environment. That's macOS protecting the system Python. The right answer is a virtual environment (venv) — a per-project sandbox where pip is allowed to install packages.

What is a venv?

A folder named .venv/ inside your project. It contains its own copy of Python and its own packages. When you "activate" it, your terminal uses that Python instead of the system one. Each project gets its own clean dependencies — no version conflicts between projects.

If you know Java / MuleSoft
A venv is like a Maven project's target/ + local repo: dependencies isolated to this project, not installed system-wide. Activating it is like loading the project's classpath.

One-time setup (run these in order)

# 1. Move into your Python project folder
cd ~/ai/claude/projects/python/

# 2. Create the venv (creates a .venv/ subfolder)
python3 -m venv .venv

# 3. Activate it — you'll see (.venv) appear in your prompt
source .venv/bin/activate

# 4. Install the SDK INSIDE the venv (now plain "pip" works)
pip install google-genai

Verify the install:

python -c "from google import genai; print('OK')"

Every time you come back to this project

You only create the venv once. But you must activate it every new terminal session:

cd ~/ai/claude/projects/python/
source .venv/bin/activate

When done, type deactivate to leave the venv.

Spotting the venv
Your shell prompt will show (.venv) at the start when active. If you don't see it, you're using system Python — pip install will fail with the externally-managed error.
Add .venv/ to .gitignore
Don't commit the venv folder — it's huge and machine-specific. Anyone else who clones your project will create their own with python3 -m venv .venv.

Tell VS Code about your venv

If VS Code still shows red squiggles on import google.genai: Cmd+Shift+P → "Python: Select Interpreter" → choose the one in ./.venv/bin/python. The squiggles will clear within seconds.

4 — The simplest possible call

Three lines of meaningful code: import, instantiate, generate.

# first_gemini.py
from google import genai

client = genai.Client()      # picks up GEMINI_API_KEY from env

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents="In one sentence, what is a MuleSoft API?"
)

print(response.text)

Run it:

python3 first_gemini.py

Expected output (will vary — that's the LLM):

A MuleSoft API is a managed integration interface that connects
applications, data, and devices using reusable connectors and flows
on the Anypoint Platform.

5 — What each line does

from google import genai

Imports the package. Note the unusual form: the install name is google-genai but the import name is google.genai — this is normal for namespaced Google packages.

client = genai.Client()

Creates a client object. With no args, it reads GEMINI_API_KEY from the environment. To pass it explicitly: genai.Client(api_key="AIza...").

client.models.generate_content(model=..., contents=...)

The actual API call. contents is your prompt — a string is the simplest form. The result has a .text attribute with the model's response as a string.

If you know Java
genai.Client() is a typical client/builder — same as instantiating HttpClient.newBuilder().build() in Java. The client.models.generate_content(...) chain is fluent-style API navigation.

6 — A more useful example

Wrap it in a function and add error handling. This is the shape you'll reuse in every agent.

# ask_gemini.py
from google import genai

client = genai.Client()

def ask(question):
    """Send a question to Gemini, return the answer text."""
    try:
        response = client.models.generate_content(
            model="gemini-3.5-flash",
            contents=question
        )
        return response.text
    except Exception as e:
        return f"Error calling Gemini: {e}"

# Try a few prompts
for q in [
    "What does the acronym REST stand for?",
    "Give me three reasons to use OAuth 2.0.",
    "Explain idempotency to a junior developer."
]:
    print(f"\nQ: {q}")
    print(f"A: {ask(q)}")
New syntax — the docstring
The triple-quoted string on the second line of ask() is a docstring — Python's built-in way to document a function. It must be the first statement inside the function. Three things make it valuable:
  • help(ask) in the REPL prints it.
  • VS Code shows it as a tooltip when you hover over ask(.
  • Frameworks like Google ADK read it to describe your function as a tool to the model. Good docstrings = good agent tools.

Java analogy: like a Javadoc /** ... */ comment — but executable and accessible at runtime.

Try it yourself
  1. Save the snippet as first_gemini.py and run it.
  2. Change the prompt to ask Gemini to summarise something you wrote (a paragraph from your own notes).
  3. Modify: Make ask() accept a second argument model_name with a default of "gemini-3.5-flash".
  4. Extend: Loop over a list of MuleSoft topics and ask Gemini to explain each one in 2 sentences. Print the results.
  5. Stretch: Add a timeout — pass config=genai.types.GenerateContentConfig(...) with the appropriate setting. (Hint: re-read your Lesson 3 cheat sheet for what timeout values to pick.)

7 — There's more on the response than just .text

The response object also tells you how many tokens you used (useful for cost estimation later):

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents="Say hi."
)

print(response.text)
print(response.usage_metadata)
# prompt_token_count: 3
# candidates_token_count: 5
# total_token_count: 8
Tip
Tokens ≈ word fragments. ~1 token per 4 characters of English. The free tier on Gemini Developer API has a generous daily quota; check the official pricing page if you start running heavier loops.

Q2. The SDK reads your API key from which environment variable by default?

  • GOOGLE_API_KEY
  • GENAI_API_KEY
  • GEMINI_API_KEY
  • API_KEY

Q3. Where should your API key live?

  • Hardcoded in the Python file so it's easy to share
  • In a comment at the top of the file
  • In an environment variable or a git-ignored .env file
  • In your README so teammates can find it

Q4. After response = client.models.generate_content(...), how do you get the model's text answer?

  • response.text
  • response.body
  • response.message
  • response.json()["text"]
