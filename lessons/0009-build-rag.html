Build Your First End-to-End RAG

Wire chunking + embeddings + retrieval + Gemini into one working RAG. The payoff lesson.

L1 ✓ L2 ✓ L3 ✓ L4 ✓ L5 ✓ L6 ✓ L7 ✓ L8 ✓ L9 — Build RAG L10 — Vector DB

What you're about to build

A script you can ask questions of, that:

  1. Loads a real Markdown file from disk
  2. Chunks it (Lesson 8)
  3. Embeds the chunks — once, then caches them to JSON (Lesson 7)
  4. Retrieves the top-K chunks for a question, with a relevance threshold
  5. Builds a grounded prompt that interleaves chunks and the question
  6. Calls Gemini with strict instructions: answer only from the provided context
  7. Returns the answer with citations — or "I don't know" when retrieval fails
Connection to your mission
This is the smallest possible RAG that works. Once you've built it, the rest of the journey (vector DB, multiple sources, hybrid search) is incremental — every upgrade replaces one piece of this skeleton.

The full pipeline

┌──── INDEX (do once) ─────┐ │ │ doc.md ─► chunk ─► embed ─► save vectors ─► chunks.json │ ┌───────────────────────────────────────────┘ │ ┌──── ASK (every question) ────┐ │ │ │ └─►│ question ─► embed ─►────► │ │ cosine search │ │ threshold check │ │ │ │ │ not enough? │ enough? │ │ ↓ ↓ │ │ "I don't build │ │ know" prompt │ │ │ │ │ ▼ │ │ Gemini call ──► │ answer + citations └──────────────────────────────┘

The new concept: grounded prompting

An ungrounded LLM call is "hey Gemini, what is OAuth?" — it answers from its training data and might hallucinate.

A grounded call is "here are 3 verified passages from our docs; answer the question using ONLY these passages; if they don't contain the answer, say so". The model becomes an *interpreter* of your retrieved context, not an oracle.

The grounding lives in two places:

  • System instruction: the rules ("answer ONLY from the context", "cite the chunk number", "say 'I don't know' if needed")
  • User message: the chunks + the question, formatted clearly
The hallucination guard
Without a strict system instruction, Gemini will try to be helpful by mixing your context with its general knowledge. That's the #1 source of subtle wrong answers. Be blunt in the system prompt: "If the context doesn't contain the answer, reply exactly 'I don't have that information.'"

Build it — step by step

Save as rag.py in your activated venv. We'll build top-down: imports, helpers, then the main flow.

Step 1 — Imports and setup

# rag.py
import json
import math
import re
from pathlib import Path

from dotenv import load_dotenv
from google import genai
from google.genai import types

load_dotenv()
client = genai.Client()

# Tunables — change these and rerun
EMBED_MODEL = "gemini-embedding-001"
CHAT_MODEL  = "gemini-3.5-flash"
DOC_PATH    = Path("anypoint-auth.md")
INDEX_PATH  = Path("index.json")
THRESHOLD   = 0.5      # cosine below this = "not relevant"
TOP_K       = 3        # how many chunks to feed the LLM
New syntax — pathlib.Path
Path is the modern, OS-agnostic way to handle file paths in Python. Beats string concatenation hands down. 
from pathlib import Path

p = Path("docs/anypoint-auth.md")
p.read_text()             # reads the file → string
p.write_text("hello")     # writes a string → file
p.exists()                # True / False
p.parent                  # Path("docs")
p.stem                    # "anypoint-auth"  (filename without extension)

Java analogy: like java.nio.file.Path + Files.readString() rolled into one fluent API.

Step 2 — Chunking (from Lesson 8)

def chunk_markdown(text):
    """Split markdown by ## headings, keeping each heading attached to its body."""
    parts = re.split(r"(?=^## )", text, flags=re.MULTILINE)
    return [p.strip() for p in parts if p.strip()]

Step 3 — Embedding helpers (from Lesson 7)

def embed(text, task_type):
    """Embed one string. task_type tells the model how this text will be used."""
    result = client.models.embed_content(
        model=EMBED_MODEL,
        contents=text,
        config=types.EmbedContentConfig(task_type=task_type)
    )
    return result.embeddings[0].values

def cosine(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    return dot / (math.sqrt(sum(x*x for x in a)) * math.sqrt(sum(y*y for y in b)))

Step 4 — Build (or load) the index

Embedding costs API calls. We don't want to re-embed every time we ask a question. Cache the chunks + their vectors to index.json; reuse it next run.

def build_index():
    """Read doc, chunk, embed, save to JSON. Returns the in-memory index."""
    print(f"Reading {DOC_PATH}...")
    text = DOC_PATH.read_text()
    chunks = chunk_markdown(text)
    print(f"Got {len(chunks)} chunks. Embedding...")
    vectors = [embed(c, "RETRIEVAL_DOCUMENT") for c in chunks]
    index = {"chunks": chunks, "vectors": vectors}
    INDEX_PATH.write_text(json.dumps(index))
    print(f"Saved index → {INDEX_PATH}")
    return index

def load_or_build_index():
    """Use cached index if it exists, otherwise build a new one."""
    if INDEX_PATH.exists():
        print(f"Loading cached index from {INDEX_PATH}")
        return json.loads(INDEX_PATH.read_text())
    return build_index()
New syntax — json.dumps / json.loads
Lesson 1 covered json.loads(string) → dict and json.dumps(dict) → string. Pairing those with Path.read_text() / Path.write_text() gives you the simplest possible way to persist Python data to disk: "convert to JSON string, write to file" on save and the reverse on load.
Cache invalidation
The cache doesn't know your doc.md has changed. If you edit the source, delete index.json before rerunning, or your old chunks will keep being served. Real systems hash the source file and re-index when the hash changes — out of scope for now.

Step 5 — Search with threshold

def search(question, index):
    """Embed the question, score every chunk, keep only those above THRESHOLD."""
    q_vec = embed(question, "RETRIEVAL_QUERY")
    scored = [(cosine(q_vec, v), c) for v, c in zip(index["vectors"], index["chunks"])]
    scored.sort(reverse=True)
    return [(s, c) for s, c in scored if s > THRESHOLD][:TOP_K]

Step 6 — The grounded prompt

This is the heart of RAG. Two pieces: system instruction (rules) and user message (context + question).

SYSTEM_INSTRUCTION = """
You are an expert assistant answering questions from a provided document.

RULES (must follow):
1. Answer ONLY from the CONTEXT below. Do not use outside knowledge.
2. If the CONTEXT does not contain the answer, reply exactly:
   "I don't have that information in the provided document."
3. Cite the chunk number(s) you used at the end of your answer, e.g. [1] [3].
4. Be concise. Maximum 4 sentences.
"""

def build_user_message(question, retrieved):
    """Format the chunks + question into one user message."""
    context_blocks = []
    for i, (score, chunk) in enumerate(retrieved, start=1):
        context_blocks.append(f"[{i}] (relevance {score:.2f})\n{chunk}")
    context = "\n\n".join(context_blocks)
    return f"CONTEXT:\n{context}\n\nQUESTION: {question}"
New syntax — "separator".join(list)
The opposite of .split(). Joins a list of strings into one string, with the separator between each: 
", ".join(["OAuth", "mTLS", "JWT"])
# → "OAuth, mTLS, JWT"

"\n\n".join(["chunk one", "chunk two"])
# → "chunk one\n\nchunk two"

Common surprise: it's a method on the separator string, not on the list. ",".join(items), not items.join(",").

New syntax — enumerate(items, start=1)
A built-in that pairs each item with its index. Cleaner than for i in range(len(items)): items[i]. 
for i, item in enumerate(["a", "b", "c"], start=1):
    print(i, item)
# 1 a
# 2 b
# 3 c

Default start=0 matches Python indexing. We use start=1 here so chunk citations read [1] [2] instead of [0] [1] (more natural in answers).

Step 7 — Ask the question

def ask(question, index):
    retrieved = search(question, index)

    # THE GUARD — refuse to retrieve when nothing is relevant
    if not retrieved:
        return "I don't have that information in the provided document."

    user_msg = build_user_message(question, retrieved)
    response = client.models.generate_content(
        model=CHAT_MODEL,
        contents=user_msg,
        config=types.GenerateContentConfig(system_instruction=SYSTEM_INSTRUCTION)
    )
    return response.text

New syntax — if __name__ == "__main__":
A common Python idiom. Code under this guard only runs when the file is executed directly (python rag.py), not when it's imported by another file (from rag import ask). Lets you build reusable modules — your test runner imports ask, but your CLI invocation runs the demo loop.

Java analogy: the same role as public static void main(String[] args) — entry point only when run directly.

The complete rag.py

The 8 steps above, assembled into one file you can copy directly. Save as rag.py in your activated venv, alongside anypoint-auth.md.

import math
import re
from pathlib import Path

from dotenv import load_dotenv
from google import genai
from google.genai import types

load_dotenv()
client = genai.Client()

# ─── Tunables ────────────────────────────────────────────────

CHAT_MODEL  = "gemini-3.5-flash"
DOC_PATH    = Path("anypoint-auth.md")
INDEX_PATH  = Path("index.json")
THRESHOLD   = 0.5      # cosine below this = "not relevant"
TOP_K       = 3        # how many chunks to feed the LLM

SYSTEM_INSTRUCTION = """
You are an expert assistant answering questions from a provided document.

RULES (must follow):
1. Answer ONLY from the CONTEXT below. Do not use outside knowledge.
2. If the CONTEXT does not contain the answer, reply exactly:
   "I don't have that information in the provided document."
3. Cite the chunk number(s) you used at the end of your answer, e.g. [1] [3].
4. Be concise. Maximum 4 sentences.
"""

Try it yourself

  2. Bigger doc: create a longer Markdown file (your own notes, a README, anything). Delete index.json and rerun. Embedding takes longer; querying stays fast.

Quick check

Q1. Why do we cache the embeddings to index.json on the first run?

  • It's required by the SDK
  • To make the cosine math more accurate
  • Embedding costs API calls and time — caching makes subsequent runs free and instant
  • It's required by Gemini's terms of service

Q2. The system instruction tells Gemini "if the context doesn't contain the answer, say so." Why is this so important?

  • It saves tokens
  • Without it, Gemini will mix retrieved chunks with its training-data knowledge — producing confident-sounding hallucinations
  • It's a legal requirement
  • It speeds up the response

Q3. What does if not retrieved: guard against in ask()?

  • Calling the LLM with no context — letting it hallucinate from its training data alone
  • A network error
  • An empty index file
  • A malformed question

Q4. If you edit anypoint-auth.md and rerun, what happens?

  • The script automatically detects the change and re-indexes
  • The cached index.json is reused — your edits are ignored until you delete the cache
  • The script crashes
  • Gemini handles the diff internally

Q5. Where do the rules for "answer only from context, cite the chunks" belong?

  • In the user message every time
  • In the system instruction — set once for all calls
  • In the embedding step
  • In the chunking function
